=== NODE TYPES ===

--------------------------------------------------------------------------------
1. INDICATOR NODES
--------------------------------------------------------------------------------
Standard technical indicators.

Type: "indicator"
Data Fields: 
- indicatorType: "SMA" | "EMA" | "RSI" | "MACD" | "BB" | "ATR" | "STOCH" | "ADX" | "ICHIMOKU" | "aroon" | "aroonosc" | "adxr" | "cmo" | "roc" | "trix" | "ultosc" | "ad" | "adosc" | "ht_trendline" | "ht_dcperiod" | "ht_phasor" | "ht_sine" | "avgprice" | "medprice" | "cdl_pattern" ...
- period: number (default 14)
- source: "close" | "open" | "high" | "low" (default "close")
- ...other specific params (stdDev, kPeriod, pattern (for "cdl_pattern"), etc.)

Example:
{
  "id": "ind_rsi",
  "type": "indicator",
  "data": { "label": "RSI", "indicatorType": "RSI", "period": 14, "source": "close" }
}

--------------------------------------------------------------------------------
2. MATH NODES
--------------------------------------------------------------------------------

Type: "math"
Data Fields:

- value: number (for "number" type)
- mathFunction: "sqrt" | "abs" | "log" ... (for "advancedMath")

Example (Add):
{
  "id": "math_add",
  "type": "math",
  "data": { "label": "Add", "mathType": "add" }

=== EDGE FORMAT ===

Edges connect nodes. The 'sourceHandle' and 'targetHandle' MUST match the node's actual handle IDs.

Handle IDs by node type:
- Indicators (default: sma, ema, rsi, atr, etc.): output 'value'
- Indicators (macd): outputs 'line', 'signal', 'histogram'
- Indicators (bb/keltner/donchian): outputs 'upper', 'middle', 'lower'
- Indicators (stochastic): outputs 'main', 'signal'
- Indicators (ichimoku): outputs 'tenkan', 'kijun', 'senkou_a', 'senkou_b', 'chikou'
- Conditions (compare/crossover/and/or): inputs 'input-a', 'input-b', output 'output'
- Conditions (not): input 'input', output 'output'
- Math/Operators: inputs 'input-a', 'input-b', output 'output'
- Math (number): output 'output'
- Environment: output 'value'
- Control (If/IfElse): input 'condition', outputs 'then', 'else'
- Control (Flow): input 'trigger', output 'output'
- Actions: input 'trigger', output 'next'
- Triggers: output 'output'
- Risk: output 'size' or 'output'

{
  "id": "edge_1",
  "source": "node_1",
  "target": "node_2",
  "sourceHandle": "value",
  "targetHandle": "input-a"
}

--------------------------------------------------------------------------------
N. DATA SOURCE NODES
--------------------------------------------------------------------------------
Optional first-class data inputs. When present, the backtest engine routes
the OHLCV fetch through the chosen provider instead of the default yfinance
path. A flow with a dataSource node automatically uses that node's symbol
and timeframe (overriding the request-level settings).

Type: "dataSource"
Subtypes (the discriminator field is `kind` for `provider == 'avanza'`
positions/watchlist; for OHLCV use `provider`):
- yfinanceData : free OHLCV (US/global stocks, ETFs, crypto)
  data: { provider: "yfinance", symbol: "AAPL", timeframe: "1d" }
- avanzaData   : Nordic broker OHLCV (auto-resolves orderbookId from ticker)
  data: { provider: "avanza", symbol: "VOLV-B.ST", timeframe: "1d" }
- fmpData      : Financial Modeling Prep daily OHLCV (needs FMP_API_KEY)
  data: { provider: "fmp", symbol: "AAPL", timeframe: "1d" }
- avanzaPositions : live portfolio positions (live-only, not backtestable)
  data: { provider: "avanza", kind: "positions" }
- avanzaWatchlist : symbols from the user's saved watchlist (live-only)
  data: { provider: "avanza", kind: "watchlist", watchlistId: "" }
- fredMacro    : St. Louis Fed time-series (CPI, unemployment, fed funds)
  data: { provider: "fred", seriesId: "CPIAUCSL" }

Example:
{
  "id": "ds_avanza",
  "type": "dataSource",
  "data": { "label": "Avanza Volvo", "provider": "avanza", "symbol": "VOLV-B.ST", "timeframe": "1d" }
}

--------------------------------------------------------------------------------
M. CUSTOM CODE NODE
--------------------------------------------------------------------------------
Drop a free-form Python or JavaScript snippet into the flow. The user (or
agent) can describe the logic in natural language and the orchestrator
fills in the `code` field via /api/ai/generate-code. Custom code is
backtest-eligible only when language is python.

Type: "llm"
Subtype: "customCode"

Example:
{
  "id": "code_macd_bias",
  "type": "llm",
  "data": {
    "label": "MACD Bias",
    "llmType": "customCode",
    "language": "python",
    "code": "def run(df):\n    return df['close'].ewm(span=12).mean() - df['close'].ewm(span=26).mean()",
    "inputs": ["ohlcv"],
    "outputs": ["number"]
  }
}
