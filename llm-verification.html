Usage

When you run a backtest with Python, Nautilus, or AI Simulation engine:

  1. Frontend calls supabase.functions.invoke('verify-backtest') with XML
  2. Gemini (via Lovable) validates the strategy structure
  3. If issues found → toast warning appears
  4. Backend runs additional verification with Gemini + DeepSeek
  5. Backtest executes with verified code

Legacy Mode (Gemini)

  • File: supabase/functions/generate-strategy/index.ts
  • Prompt: 2770-line FULL_SYSTEM_PROMPT.txt
  • Model: google/gemini-3-pro via Lovable

Default Mode (DeepSeek)

  • File: backend/main.py → generate_strategy()
  • Prompt: GCG_PLANNER_PROMPT + SYSTEM_PROMPT
  • Model: deepseek-chat + deepseek-reasoner
