Code Resolution Priority

When a backtest is requested, the backend checks these sources in order:

  1. Template ID → Load pre-built Python from template_strategies.py (uses talib)
  2. Strategy ID → Load cached code by ID (from AI generation)
  3. Exact XML Hash → Load cached code by exact XML match
  4. Structure Hash → Load cached code with same blocks, inject new params
  5. Local Generation → Parse XML → Generate Python → Save to cache
  6. LLM Generation → If local fails, use DeepSeek to fix/complete code

Generation Outputs

AI generation now produces both XML and Python code:

  • XML: Blockly workspace for visual editing
  • Python: Executable strategy code for backtesting.py
  • Strategy ID: Unique identifier for cache lookup

Structure Hash Comparison

The system compares XML structure (ignoring numeric values) to detect parameter-only changes:

Original XML

<field name="NUM">14</field>

Structure Hash

<field name="NUM">#</field>

Same structure hash = Only numbers changed → Reuse cached Python code with new params
Different structure hash = Blocks added/removed → Full regeneration needed

Verification Logic

Code Source Verification Reason
Template (pre-built) SKIP ✓ Pre-tested talib code
Cached by ID SKIP ✓ Already verified on first run
Structure cache + params SKIP ✓ Same code, different numbers
Fresh generation RUN ⚠ New code needs validation
